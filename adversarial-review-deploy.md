1. **[high]** `ensure_workspace_prompts_inner` treats directory existence as proof that the workspace root deployment is healthy (`app/src-tauri/src/commands/workflow/deploy.rs:235-238`), then uses that root as the source of truth for every per-skill deploy (`app/src-tauri/src/commands/workflow/deploy.rs:266-294`). If `<workspace>/.agents/agents/skill-creator.md` or any bundled AgentSkill under `<workspace>/.agents/skills/` is deleted or left half-written after a prior run, tier 1 will not repair it because `source_sha` still matches and `layout_ok` only checks the directories. Tier 2 then hashes the corrupted root and fans that broken state out to every `<workspace>/<plugin>/<skill>/.agents` by clearing and recopying from the incomplete root (`app/src-tauri/src/commands/workflow/deploy.rs:395-458`). One missing root file becomes a workspace-wide runtime outage instead of being healed from bundled sources.
2. **[medium]** The sync entrypoint leaves the optimistic cache poisoned on copy failure. `ensure_workspace_prompts_inner` stores the new `source_sha` before any filesystem copy succeeds (`app/src-tauri/src/commands/workflow/deploy.rs:239-246`), but only the async wrapper clears that cache on error (`app/src-tauri/src/commands/workflow/deploy.rs:198-204`). `ensure_workspace_prompts_sync` just returns the error (`app/src-tauri/src/commands/workflow/deploy.rs:304-315`). A startup deploy that fails after recreating `.agents/agents` and `.agents/skills` but before repopulating them will leave cached SHAs pointing at a partial deployment; any later in-process sync retry will see matching SHA plus existing directories and skip the repair.
3. **[low]** This change pays a full tree hash plus workspace scan on every workflow/refine/eval dispatch (`app/src-tauri/src/commands/workflow/deploy.rs:40-63`, `app/src-tauri/src/commands/workflow/deploy.rs:266-295`, `app/src-tauri/src/commands/refine/mod.rs:249-253`, `app/src-tauri/src/commands/workflow/runtime.rs:631-633`, `app/src-tauri/src/commands/eval_workbench/mod.rs:1225-1228`) to solve a single dev-mode problem: live pickup of edits under `agent-sources/workspace/**` without restarting the app. In production the bundled sources are immutable, and `clear_workspace` already exists as the explicit redeploy path (`app/src-tauri/src/commands/workspace.rs:413-417`). The extra SHA cache, global mutex state, and per-dispatch I/O are complexity costs with no concrete second use case, and they are what make the corruption paths above possible.
