## Summary

This branch refactors the Eval Workbench into an authoring-only surface: scenarios and assertions are stored in SQLite, generation uses the selected-skill OpenHands conversation, and all app-owned eval-run execution concepts are removed. 9 commits, rebased onto main.

The implementation is thorough and well-scoped. All 8 acceptance criteria are either proven or have only minor documentation-level findings. Validation commands pass cleanly (Rust 8/8, frontend 52/52, contract 81/81, repo-map audit passed, TypeScript clean, markdownlint clean).

One medium finding: `repo-map.json` description text still references deleted files (`eval-run.ts`, `eval-running-state.ts`, `description/` module). The automated repo-map audit passes because it checks path existence, not description text accuracy, but the stale descriptions violate AC 7's intent.

## Findings

### Medium

1. **[Architect]** `repo-map.json` description text references deleted files. Lines 106 and 110 describe `eval-run.ts`, `eval-running-state.ts`, and `description/` module, all of which are deleted. The automated audit passes because it checks path existence, not description text. Recommendation: update `frontend_lib` and `rust_commands` description strings to reflect the clean-break model.

### Low

1. **[Minimalist]** `eval-workbench.ts` still exports `scenarioNameSlug()` (line 27-33), which was used for YAML file path generation. It is no longer called anywhere in the eval path. Not harmful, but dead code. Recommendation: remove if unused, or leave as a utility if it has other consumers.

## What Went Well

1. **Clean deletion discipline.** All files on the explicit delete list (`scenarios.rs`, `eval-run.ts`, `eval-running-state.ts`, `eval-run.test.ts`, eval-running guards) are confirmed removed with no orphan imports.
2. **Generation error handling.** The "no active conversation" error message is user-facing and actionable ("Select a skill conversation in Refine first."), matching the pattern Workflow uses.
3. **Race condition fix.** The `setup_turn_listeners` → `send_openhands_message` → `wait_for_turn_result` ordering (commit `cda68c60`) correctly sets up listeners before dispatching, avoiding the classic event-listener race.

## Verdict

**APPROVE with minor changes**

All 8 acceptance criteria are proven with code evidence. The only finding is stale `repo-map.json` description text (AC 7), which is a documentation accuracy issue, not a functional defect. All validation commands pass cleanly.
