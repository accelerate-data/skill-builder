Implementation Notes

Agent modes

  • Mode 1 — generate eval: receives intent + skill context → writes pending-eval.json (name, slug, prompt, expectations)
  • Mode 2 — re-generate from intent: receives updated intent + skill context → rewrites pending-eval.json entirely (same file)

Frontend state (workspace-evals.tsx)

  • intentOpen — controls Screen 1 intent dialog visibility
  • isGenerating / isRegenerating — controls Screen 2 / Screen 4 loading states
  • reviewOpen — controls Screen 3/4 review dialog visibility
  • draftEval — in-flight eval being reviewed/edited (includes intent field)
  • evalQueue: TestCase[] — pending evals (Screen 5); managed in local state, not Zustand

New Rust commands

  • Existing read_pending_eval / discard_pending_eval remain unchanged — single-eval per agent run

Skill changes (SKILL.md)

  • Add Mode 1 / Mode 2 distinction driven by prompt structure
  • Add references/pluggable.md for path resolution (like skill-description-optimizer)
  • Reference Test Cases section of skill-creator/SKILL.md for eval quality guidance
