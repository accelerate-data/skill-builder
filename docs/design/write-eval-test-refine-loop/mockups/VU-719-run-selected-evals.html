VU-719: Run Selected Evals

Select test cases from evals.json, choose 1× or 3× runs for statistical confidence, view live grading progress, get a benchmark + analyst notes, and optionally refine the skill.

State 1 — Select Evals & Run Count

Prompts are already stored in each test case (evals.json). User selects which cases to run and how many repetitions for statistical reliability.

Implementation Notes

Agent: evaluate-skill

  • Located at agent-sources/plugins/skill-creator/agents/evaluate-skill.md
  • Inputs: skill_name, workspace_path, eval_ids[], run_count (1|3), skill_path
  • Emits {"type":"eval_graded", "runIndex":…, "evalIndex":…, "totalEvals":…, "grading":{…}} after each graded eval
  • Emits {"type":"complete", "iteration":N, "benchmark":{…}, "analyst_notes":[…]} when done
  • Writes: iteration-N/benchmark.json, iteration-N/analyst-notes.json, per-eval grading.json

Rust command

  • New command: run_selected_evals(skill_name, workspace_path, eval_ids[], run_count)
  • Dispatches evaluate-skill agent via sidecar pool
  • Emits eval-run-progress Tauri event per eval_graded structuredOutput received
  • Returns final BenchmarkResult on complete structuredOutput

State management

  • Component-local (useState): runCount: 1|3, selectedIds: number[], isRunning, gradedCount, analystNotesOpen
  • Zustand store: benchmark + analystNotes — persists across tab navigation so Refine tab can pre-fill its message
  • On "Refine skill" click: write prefillMessage to refine store, navigate to Refine tab

Acceptance criteria

  • Run count toggle switches between 1× and 3× (stored in local state)
  • Checkboxes on eval rows; "Run selected (N)" button disabled when N=0
  • No prompt input — each eval uses its own stored prompt from evals.json
  • Progress banner updates on each eval_graded event
  • Status dot for in-progress eval pulses; queued evals shown at reduced opacity
  • Benchmark card shows aggregate stats + per-eval pass rates after completion
  • 3× mode shows per-run pills (R1/R2/R3) in eval breakdown rows
  • Analyst notes auto-expand when has_failures is true
  • Refine CTA hidden when has_failures is false; visible and enabled otherwise
  • Clicking "Refine skill" navigates to Refine tab with pre-filled message
  • Benchmark + analyst_notes stored in Zustand so they survive tab navigation
