Implementation Notes

Run Mode Toggle

  • State: runMode: '1' | '3' in eval-tab.tsx
  • 1-run: single <textarea> for prompt; 3-run: three labelled textareas (Run 1/2/3)
  • Auto-generate button calls generate_eval_prompts(skillName, workspacePath, count) → returns string[]
  • Generated prompts pre-fill the textarea(s); user can edit before running

Execution

  • New Rust command: run_selected_evals(skill_name, workspace_path, eval_ids[], prompts[])
  • Runs sequentially: Run 1 completes before Run 2 starts (avoids resource contention)
  • Each run result stored in a new iteration-N/ subdirectory
  • Progress events emitted via Tauri events: eval-run-progress → { runIndex, testIndex, total }

Results Display

  • Collapsed by default; user clicks chevron to expand a run's per-assertion detail
  • 3-run aggregate: avg pass rate = sum(pass) / total assertions across all runs
  • Test list dots update live as each run's result arrives
  • Score shown as passed/total (e.g. 7/12) in run header

Rewrite CTA

  • Visible only when totalFailures > 0 after all runs complete
  • Clicking switches to Refine tab: sets prefillMessage in refine store
  • Pre-fill format: list of failing assertions grouped by test, with run counts
  • Message is editable in the Refine input before sending

Files

  • app/src/components/workspace/eval-tab.tsx — extends VU-718 component
  • app/src-tauri/src/commands/workflow/evaluation.rs — add run_selected_evals
  • app/src/lib/tauri.ts — add runSelectedEvals wrapper
  • app/src/lib/types.ts — add EvalRunResult, RunProgress
  • app/src/stores/refine-store.ts — add prefillMessage field

Acceptance Criteria

  • Run mode toggle switches between 1-prompt and 3-prompt input
  • Auto-generate pre-fills prompt field(s) with domain-relevant scenarios
  • Checkboxes on test rows; "Run selected (N)" button counts checked rows
  • Run button disabled when no tests selected or prompts empty
  • Progress indicator updates as each run/test completes
  • Results shown as collapsible run groups (Run 1, 2, 3)
  • Per-assertion pass/fail shown when run group is expanded
  • Aggregate pass rate shown in summary bar
  • "Rewrite in Refine" CTA visible only when failures exist
  • Clicking Rewrite navigates to Refine tab with pre-filled message
  • Pre-filled message lists failing assertions and can be edited
