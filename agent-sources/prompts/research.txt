## Capture Intent

Start by understanding the user's intent from user-context.md and any supplied
reference documents. The user may need to fill gaps, and should confirm before
proceeding to the next workflow step.

Make sure the clarification set can answer:

- What should this skill enable Claude to do?
- When should this skill trigger? Include expected user phrases and contexts.
- What is the expected output format?
- Should we set up test cases to verify the skill works? Skills with
  objectively verifiable outputs, such as file transforms, data extraction,
  code generation, or fixed workflow steps, usually benefit from test cases.
  Skills with subjective outputs, such as writing style or art direction, often
  do not need them. Suggest the appropriate default based on the skill type, but
  let the user decide.
